# AgentRadar 5 分钟演示脚本

## 演示前准备

```bash
docker compose up --build -d
docker compose exec backend python -m app.demo
docker compose exec backend python -m app.evaluation evaluation/agent_cases.json --format markdown
```

确认 <http://localhost:8080> 可以打开，并预先在 `.env` 配置可用 GitHub Token。没有网络或 Token 时仍可完整展示热门雷达、离线评测和架构；智能搜索部分改用已经录制的备用画面。

## 0:00—0:40 项目定位

打开首页，说明 AgentRadar 解决的是 GitHub Agent 项目搜索噪声、README 可信度不足和陌生仓库阅读入口不清晰的问题。强调两个入口：自然语言智能搜索与基于历史快照的热门项目雷达。

## 0:40—2:40 智能搜索闭环

输入：

```text
帮我找适合 Python 后端开发者学习的 LangGraph 项目，
最好有 FastAPI、工具调用和状态管理，
不要简单聊天套壳，适合改造成简历项目。
```

依次展示：

1. 候选发现、规则过滤、研究目标和最终推荐数量；
2. 推荐卡的六维评分、匹配理由、优点、缺点与套壳风险；
3. 展开真实阅读路径，说明路径只能来自 GitHub 目录树；
4. 执行轨迹中的节点耗时和 GitHub 工具；
5. 收藏第一名并忽略一个不需要的项目；
6. 追加“只保留最近半年更新、不要 CrewAI、优先简单项目”；
7. 说明系统复用当前候选和分析报告，没有从头搜索。

## 2:40—3:40 热门雷达

切换“今日热门”“本周上升”“新项目潜力”，展示：

- 24 小时与 7 天 Star 增量；
- 热度分、质量分和 Agent 能力完整度分开呈现；
- 快照置信度和升温原因；
- LangGraph、Agent Memory、MCP 等分类。

打开“显示演示数据”开关，指出演示卡片带有醒目标识且默认不会混入真实榜单。关闭开关后展示真实采集状态，并说明日榜和周榜分别需要 24 小时、7 天的历史窗口。

## 3:40—4:25 工程与可信度

打开 API 文档或架构图，说明：

- LangGraph 负责节点编排，规则负责低成本硬过滤；
- GitHub 工具只读，并具有缓存、重试、超时和响应裁剪；
- PostgreSQL 保存仓库、快照、分析报告、会话、轨迹和收藏；
- 单个仓库失败不会让整次搜索失败；
- 陌生仓库永远不会在服务端自动执行。

## 4:25—5:00 评测与总结

展示 `docs/evaluation.md`：固定需求字段准确率 100%、错误项目过滤召回率 100%、不存在文件路径幻觉率 0%。最后总结：项目重点不是“再做一个 GitHub 搜索页”，而是用可控工作流把发现、调查、证据、评分、趋势和用户反馈连成闭环。

## 录制检查清单

- 浏览器缩放设为 90%—100%，避免卡片内容被裁切；
- Token、数据库连接串和终端历史不要出现在画面中；
- 先录一份智能搜索成功结果作为断网备用；
- 录制时长控制在 5 分钟，720p 以上；
- 画面至少覆盖搜索、详情、收藏、继续筛选、热门榜单、轨迹和评测报告。
